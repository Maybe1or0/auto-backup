\section{Architecture}
L'architecture est organisee autour d'un noyau de scripts Bash modulaires et d'un
ensemble de fichiers de configuration. Les principaux dossiers sont :

\begin{itemize}
  \item \texttt{config/} : variables d'execution (utilisateur, repertoire de
  sauvegarde, mode de clonage).
  \item \texttt{src/} : modules pour la journalisation, l'acces a GitHub et les
  operations Git.
  \item \texttt{systemd/} : unite \texttt{.service} et timer \texttt{.timer}.
  \item \texttt{logs/} : traces d'execution centralisees.
  \item \texttt{doc/} : documentation LaTeX.
\end{itemize}

Le script principal orchestre les modules : verification des prerequis, collecte
liste des depots, clonage ou mise a jour, puis journalisation de chaque action.
